## 🧪 功能测试清单

### ✅ Phase 1: 插件系统测试

#### 1.1 测试插件加载器
1. 启动应用
2. 点击侧边栏的 **🧩 Plugins** 图标
3. 应该看到插件管理界面

#### 1.2 测试示例插件
1. 在插件界面中找到 "Example Plugin"
2. 点击 **Load** 按钮加载插件
3. 检查控制台是否有日志：
   ```
   [PluginLoader] Loading plugin: example-plugin
   [ExamplePlugin] Initialized
   ```
4. 点击 **Unload** 按钮卸载插件
5. 检查控制台日志：
   ```
   [PluginLoader] Unloading plugin: example-plugin
   [ExamplePlugin] Disposed
   ```

#### 1.3 测试插件配置
1. 打开浏览器开发者工具（F12）
2. 在 Console 中运行：
   ```javascript
   // 检查插件注册表
   window.__DESK_DEBUG__ = true;
   ```
3. 重新加载插件，观察配置是否正确加载

#### 1.4 测试事件系统
1. 在 Console 中运行：
   ```javascript
   // 监听插件事件
   window.addEventListener('desk:plugin:loaded', (e) => {
     console.log('Plugin loaded:', e.detail);
   });
   ```
2. 加载/卸载插件，检查事件是否触发

## 🐛 常见问题排查

### 问题 1: TypeScript 编译错误

**症状**: `npx tsc --noEmit` 报错

**解决方案**:
```bash
# 检查是否缺少类型定义
npm install --save-dev @types/node @types/react @types/react-dom

# 检查 tsconfig.json 配置
cat tsconfig.json
```

### 问题 2: 模块导入错误

**症状**: `Cannot find module '@/...'`

**解决方案**:
检查 `vite.config.ts` 或 `tsconfig.json` 中的路径别名配置：
```typescript
// vite.config.ts
resolve: {
  alias: {
    '@': path.resolve(__dirname, './src'),
  },
}
```

### 问题 3: Zustand 状态不更新

**症状**: 状态修改后 UI 不刷新

**解决方案**:
1. 检查是否正确使用 Zustand hooks
2. 检查 Redux DevTools 中的状态是否更新
3. 确保组件正确订阅了状态

### 问题 4: 命令面板不显示

**症状**: 按 Ctrl+Shift+P 没有反应

**解决方案**:
1. 检查快捷键是否被其他程序占用
2. 检查 Console 是否有错误
3. 确认命令系统是否正确初始化

### 问题 5: 插件加载失败

**症状**: 插件无法加载

**解决方案**:
1. 检查插件路径是否正确
2. 检查插件配置是否有效（Zod 验证）
3. 查看 Console 中的错误信息
